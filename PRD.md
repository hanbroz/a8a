# 프로젝트 개요

저는 대한민국 저가항공(LCC) 이스타항공에서 여행사(OTA)에게 API를 제공하고 관리하는 업무를 하고 있습니다.

여행사들에게 API에 대하여 안내하고 문제가 생기면 같이 문제를 해결하는 업무를 하고 있습니다.

이스타항공은 Navitaire PSS를 사용하고 있고 Navitraire에서 제공하는 API를 중간에서 게이트웨이 형태로 여행사들에게 제공하고 있습니다.

그래서, 하루에도 여러번 API 기능을 테스트하기 위해 API 테스트를 하고 있습니다.

이런 API는 서로 기능이 연계되어 있으며, 이러한 기능은 토큰발급 부터 예약, 발권, 환불 등으로 이어지는 일련의 과정이 있습니다.

이러한 end-point는 연계하기 위해서는 선행되는 API의 응답값의 특정 부분을 다음 API에 요청으로 넘겨주어야 하는 경우가 많습니다.

예를 들어, 예약 API의 응답값에서 예약번호를 추출하여 발권 API에 요청으로 넘겨주는 경우가 있습니다.

이렇게 응답에서 받은 값을 다음 API의 요청으로 넘겨주는 과정을 수동으로 하려면, 응답값에서 필요한 부분을 찾아서 복사하고, 다음 API의 요청에 붙여넣는 작업이 필요합니다.

이러한 번거러운 작업을 n8n과 같은 워크플로우 자동화 도구를 만들어서 자동화하려고 합니다.

## 처리 프로세스 개요 (예시)

1. 고개사라는 데이터 모듈을 구성합니다. 데이터 모듈은 key-value 형태의 json 형태로 구성됩니다.
2. API 모듈을 구성합니다. 이 API는 독립적인 헤더, 인증, body, 메서드등 http(s)가 가져야 할 모든 요소를 가지고 있습니다.
3. 먼저 토큰 발급 이라는 API 모듈을 만들고 이 모듈을 앞에 고객사 데이터 모듈에 INPUT으로 연결 하고 중간에 셀렉터 모듈을 배치합니다. 셀렉터 모듈은 고객사 데이터 모듈에서 토큰 발급 API 모듈로 어떤 데이터를 넘겨줄지 선택하는 역할을 합니다.
4. 다음은 여정검색 이라는 API 모듈을 만들고 토큰의 발급 API의 응답 값을 여정검색의 요청 모듈의 헤더에 토큰값을 넘겨주는 형태로 연결합니다.
5. 다음은 예약 API 모듈을 만들고 여정검색 API의 응답 값을 예약 API의 요청 모듈의 body에 필요한 부분을 넘겨주는 형태로 연결합니다. 이때 여정검색의 응답의 특정 값을 선택할 수 있도록 중간에 셀렉터를 배치 합니다.

- 이렇게 데이터 모듈, 셀렉터, API 모듈을 조합하여 API의 연계 과정을 자동화할 수 있습니다.
- API 호출되는 일련의 API 요청과 응답을 기록하여 고객사에게 API의 연계 과정이 어떻게 진행되는지 보여줄 수 있습니다. 이를 통해 고객사는 API의 연계 과정을 쉽게 이해할 수 있습니다.

## 개발 환경 및 요구사항

### 필수 선행 조건 (절대 규칙)

- **디자인 우선 원칙**: 개발 착수 전 반드시 Claude Design 도구를 통해 전체 UI/UX 화면을 설계하고 승인받아야 합니다. 
Fetch this design file, read its readme, and implement the relevant aspects of the design. https://api.anthropic.com/v1/design/h/y7IUCOVsDpMecTnAoFDVeg?open_file=A8A+Workflow.html, Implement: A8A Workflow.html, (IMPORTANT: 디자인을 한꺼번에 반영하지 마십시오. 레이아웃을 우선 구성하고 요소는 지시사항에 따라 하나 하나 씩 구성합니다.)
- **최신 LTS 버전 확인 의무**: 개발 전 반드시 Context7 MCP를 사용하여 Electron, Node.js, React, TypeScript 등 사용할 모든 라이브러리와 런타임의 최신 LTS 버전을 실시간으로 확인합니다. 학습 데이터나 기억에 의존한 버전을 사용하지 않으며, 반드시 Context7 MCP 조회 결과를 근거로 버전을 결정합니다.

### 개발 환경

- 윈도우와 맥에서도 모두 사용할 수 있도록 Electron 프레임워크를 사용하여 개발합니다.
- 각 API 구성은 파일로 저장하여, 사용자가 필요에 따라 API 구성을 쉽게 불러오고 수정할 수 있도록 해야 합니다.
- 모든 모듈은 자원(Asset)으로 관리되어야 하며, 사용자가 자원을 쉽게 추가, 수정, 삭제할 수 있도록 해야 합니다.
- 사용되는 모든 버전의 최신의 LTS 버전을 사용하여 개발합니다.
- 절대 Deprecated된 버전이나 기능 메서드를 사용해서는 안됩니다.
- API의 연계 과정에서 발생할 수 있는 오류나 예외 상황을 적절히 처리하여 안정적인 동작을 보장해야 합니다.
- 사용자 인터페이스는 직관적이고 사용하기 쉬워야 합니다. API의 연계 과정을 시각적으로 보여주는 기능을 포함하여, 사용자가 쉽게 이해할 수 있도록 해야 합니다.
- API의 연계 과정에서 발생하는 모든 요청과 응답을 기록하여, 사용자가 API의 연계 과정을 쉽게 추적할 수 있도록 해야 합니다. 이를 통해 사용자는 API의 연계 과정에서 발생하는 문제를 쉽게 파악하고 해결할 수 있도록 해야 합니다.
- 개발된 기능을 쉽게 실행할 수 있도록 bat, sh 스크립트를 제공하여, 사용자가 명령어 한 줄로 기능을 실행할 수 있도록 해야 합니다. 이를 통해 사용자는 개발된 기능을 쉽게 활용할 수 있도록 해야 합니다.
- 개발된 기능은 API의 연계 과정을 자동화하는 데 중점을 두어야 하며, 사용자가 API의 연계 과정을 쉽게 이해하고 활용할 수 있도록 해야 합니다. 이를 통해 사용자는 API의 연계 과정을 효율적으로 관리할 수 있도록 해야 합니다.
