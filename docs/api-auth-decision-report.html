API 인증 기능 의사결정 보고

API 모듈에 Auth 탭을 추가하고 선택한 인증 값이 테스트 실행, 미리보기 실행, 전체 워크플로 실행에 반영되도록 구현했습니다. 구현 대상은 Bearer, Basic, OAuth 2.0, API Key로 한정했습니다.

결정 사항

항목 결정 사유
Bearer Authorization: Bearer ... 헤더를 자동 구성합니다. 토큰 값만으로 기존 API 호출 직전에 적용할 수 있습니다.
Basic 아이디와 비밀번호를 Base64로 인코딩하여 Authorization: Basic ... 헤더를 구성합니다. 추가 통신이나 챌린지 처리가 필요하지 않습니다.
OAuth 2.0 Access Token을 Authorization: Bearer ... 헤더로 적용합니다. 토큰 발급 플로우는 포함하지 않고, 사용자가 입력한 토큰을 호출에 반영합니다.
API Key Header 또는 Query Param 중 선택한 위치에 키와 값을 적용합니다. 서비스별 API Key 전달 방식 차이를 수용하기 위해 위치 선택을 제공합니다.
제외 방식 Digest, NTLM, OAuth 1.0, WSSE, AWS Sig v4, Inherit는 드롭다운과 호출 로직에서 제외했습니다. 현재 구현 범위를 네 가지 인증 방식으로 제한하기로 했기 때문입니다.
템플릿 변수 Auth 입력 값에서도 {{환경변수}}와 [[INPUT]] 변수를 사용할 수 있도록 했습니다. URL, Header, Param, Body와 동일한 변수 치환 흐름을 유지하기 위해서입니다.

검증

npx tsc -b --noEmit와 npm run build를 실행하여 타입 검사와 프로덕션 빌드를 통과했습니다.
