의사결정 사항

모듈 삭제 정책

결정: 현재 UI 문구와 구현 방향을 유지해, 사이드바에서 모듈 원본을 삭제하면 해당 모듈을 사용하는 캔버스 노드 인스턴스도 함께 삭제합니다. 대신 DB에서 관련 엣지를 먼저 삭제해 고아 연결선을 남기지 않도록 했습니다.

워크플로우 연결 모델

결정: 현재 실행기는 단일 입력 노드 흐름을 기준으로 동작하므로, 노드 하나에 입력 연결 하나만 허용합니다. 다중 입력 DAG 실행 모델은 후속 기능으로 분리합니다.

API 실패 후 리포트 생성

결정: API 노드가 실패하면 이후 업무 노드는 중단하되, 연결된 End 노드로 이동해 실패 로그가 포함된 리포트를 생성할 수 있게 했습니다.

사용자 스크립트 격리

보류: Pre/Post 스크립트는 현재 로컬 신뢰 실행 모델을 유지합니다. Worker 또는 별도 샌드박스 격리는 보안 설계가 필요한 후속 과제로 남겼습니다.

계획 HTML 문서 처리

보류: 기존 계획 문서는 삭제하지 않았습니다. 대신 현재 아키텍처와 기술부채 문서를 최신 상태로 고쳤고, 계획 문서의 archive 이동 여부는 별도 정리 작업으로 남겼습니다.
