1요약 (Executive Summary)

API 모듈에 Pre Request와 Post Response 두 시점의 JavaScript 스크립트를 추가합니다. Postman·Bruno의 동일 기능을 이 앱의 워크플로우 컨텍스트에 맞게 이식하는 작업입니다.

  • Pre Request: API 호출 직전 1회 실행 — INPUT을 가공해 [[변수]]로 주입
  • Post Response: API 응답 직후 ~ 다음 모듈 전달 직전 — 응답에서 값을 추출해 {{환경변수}} 영구 저장 또는 [[변수]]로 다음 모듈에 전달
  • 에디터: Monaco (VSCode 엔진) — 라인번호·구문강조·자동완성
  • 레이아웃: 좌측 컬럼은 INPUT(위) + Pre Script(아래), 우측은 OUTPUT(위) + Post Script(아래), 가운데는 기존 설정

2합의된 설계 결정

이전 대화에서 사용자 승인 완료된 항목입니다.

스크립트 두 시점을 어떻게 배치할까?
Pre/Post 두 영역을 위·아래로 분리. 탭 전환이 아니라 동시에 보이는 두 패널.
Pre Request 스크립트의 실행 단위는?
노드 실행당 1회. INPUT이 배열이어도 스크립트는 한 번만 실행되고 getInput()은 전체 배열을 반환.
Post Response 스크립트에서 setInput()을 호출하면?
에러로 차단. 시점별로 호출 가능한 함수만 노출 — 혼란 방지.
Bruno의 "Tests" 탭(assertion)도 포함?
이번에는 제외. Pre/Post 두 시점만 먼저 완성.
스크립트 API 패러다임은?
Bruno 방식 — 함수 호출 기반. 스크립트 내부에서 getInput()으로 값을 참조하고, setEnv() / setInput() / setOutput() 호출로 변수를 설정. 객체를 return하는 방식은 사용하지 않음.
코드 에디터 라이브러리?
Monaco (VSCode·Postman·Bruno와 동일 엔진). 다크 테마 vs-dark, JavaScript 언어 모드.

3UI 레이아웃 시각화

모달 본문 영역을 3컬럼으로 유지하되, 좌·우 컬럼만 세로 분할합니다. 모든 분할선은 마우스 드래그로 비율 조정 가능.

▶ INPUT (JSON)
[
  { "userName": "TRP_IOTA", ... }
]
▼ PRE REQUEST · JavaScript
// API 호출 전 1회
const u = getInput()[0];
setInput('token', btoa(...));
⚙ 설정 (URL · Headers · Params · Body)
POST [[token]] / https://api/{{host}}
Content-Type: application/json
body: { "domain": "[[domain]]" }
▶ OUTPUT (JSON)
{
  "data": { "token": "eyJ..." }
}
▼ POST RESPONSE · JavaScript
// 응답 처리 1회
const t = getOutput().data.token;
setEnv('AUTH_TOKEN', t);

↕ 가로 분할선 · ↔ 세로 분할선 — 모두 마우스로 비율 조정

4스크립트 API 명세

4-1. 함수 가용성 매트릭스

함수Pre RequestPost Response반환·동작
getInput() ✔ 사용 가능 ✔ 사용 가능 이전 모듈의 OUTPUT을 그대로 반환 (JSON 파싱 완료). 타입이 객체이면 객체로, 배열이면 배열로 그대로 노출.
예) 이전 모듈이 { "name": "lee dohan", "age": 21 } 을 보냈다면 getInput()는 그 객체 자체.
getOutput() ✘ throw "Pre에서는 getOutput() 사용 불가" ✔ 사용 가능 API 응답 전체 반환 (파싱된 JS 객체)
setEnv(name, value) ✔ ✔ {{name}}에 영구 저장. 활성 환경의 변수에 upsert.
setInput(name, value) ✔ ✘ throw "Post에서는 setInput() 사용 불가" 이 노드의 URL/Headers/Body interpolation에서 [[name]]으로 사용. 노드 실행 종료 시 휘발.
setOutput(name, value) ✘ throw ✔ OUTPUT에 변수 부착. 다음 모듈의 [[name]]으로 사용 가능. 다음 노드 실행 종료 시 휘발.
console.log / warn / error ✔ ✔ DevTools 콘솔에 출력 (디버깅용)

4-2. 값 타입

  • 문자열: 그대로 저장. interpolation 시 원문 삽입.
  • 객체 / 배열: 그대로 저장. interpolation 시 JSON.stringify(value) 자동 적용.
  • 숫자 / boolean: String(value)로 변환 후 삽입.

4-3. JSON 탐색

일반 JavaScript 문법 그대로 사용 가능합니다:

const output = getOutput();
const token = output.data.access_token;            // 객체 필드
const firstId = output.items[0].id;             // 배열 인덱스
const emails = output.users.map(u => u.email);  // 메서드
const active = output.users.filter(u => u.active); // 체이닝

setEnv('AUTH', token);
setOutput('firstUser', active[0]);            // 객체 저장 OK

4-4. 비동기 지원

스크립트는 AsyncFunction으로 래핑되므로 await가 자유롭게 사용 가능합니다.

// Pre Request에서 다른 API 호출 후 토큰 발급도 가능
const res = await fetch('https://auth/token');
const { token } = await res.json();
setEnv('TOKEN', token);

5실행 모델 & 데이터 흐름

5-1. 단일 API 노드 실행 시퀀스

1. 노드 진입 — 상위 모듈로부터 inputArray 수신
2. Pre Request 스크립트 (있으면) — 노드당 1회
   ├── getInput() → 전체 inputArray
   ├── setInput(k, v) 수집 → preInputVars 맵
   └── setEnv(k, v) → 즉시 DB upsert + envVarsForExec 갱신

3. 각 INPUT row마다 fetch 루프
   for row of items:
     item = { ...row, ...preInputVars }   // script 값이 우선
     url     = resolveTemplate(cfg.url, env, item)
     headers = resolveTemplate(cfg.headers, env, item)
     body    = resolveTemplate(cfg.body, env, item)
     res     = await window.api.http.fetch(url, ...)
     allResults.push(parsed)

4. Post Response 스크립트 (있으면) — 노드당 1회
   ├── getInput() → 전체 inputArray
   ├── getOutput() → allResults (배열)
   ├── setEnv(k, v) → DB upsert
   └── setOutput(k, v) → postOutputVars 맵

5. 결과 확정
   nodeOutputs[nodeId] = allResults
   moduleVars[nodeId]  = postOutputVars   // ★ 신규 맵

5-2. 다음 모듈로의 변수 전파 — [[name]] 해석 규칙

현재 모듈(Pre Request 스크립트 / URL / Headers / Params / Body 어디서든)에서 [[name]]을 해석할 때 다음 순서로 찾습니다.

우선순위출처비고
1 (최우선) moduleVars[upstreamNodeId] 이전 모듈의 setOutput(name, value) 결과
2 현재 노드의 preInputVars 이 노드의 Pre Request에서 setInput(name, value)로 주입한 값
3 이전 모듈의 OUTPUT 객체 필드 upstream output이 객체면 그 객체의 name 키.
upstream output이 배열이면 현재 iteration row의 name 키.
4 (해석 실패) literal로 남김 [[name]] 그대로 출력
핵심: upstream 객체 / 배열 모두 동일하게 동작
이전 모듈이 { "name": "lee dohan", "age": 21 } 한 객체를 보내든, [{ "name": "lee dohan", "age": 21 }] 배열을 보내든 [[name]] = "lee dohan" 으로 동일하게 해석됩니다. 구현 측에서 단일 객체는 1-element array로 정규화한 뒤 row 처리에 진입하기 때문.
설계 결정 — 우선순위의 의미
명시적으로 스크립트가 설정한 값(setOutput · setInput)이 raw 데이터(upstream object/row)보다 우선합니다. 사용자가 코드로 의도를 표현한 것은 자동 매핑보다 강한 신호이기 때문.

5-3. 에러 처리

  • 스크립트 throw → OUTPUT 패널 하단에 빨간 상태바 (메시지 + 라인번호)
  • 해당 노드 실행 중단 → nodeStatuses[nodeId] = 'error'
  • 다음 모듈로 진행하지 않음 (기존 fetch 실패와 동일 흐름)

7사용자 시나리오 예제

시나리오 A: Basic 인증 토큰을 응답에서 받아 다음 모듈로 전달

Pre Request 스크립트:

// INPUT에서 userName/password 추출 → Basic 인증 헤더 만들기
const { userName, password } = getInput()[0];
const basic = btoa(`${userName}:${password}`);
setInput('authHeader', `Basic ${basic}`);

설정 (가운데 컬럼):

POST  http://api.example.com/login
Authorization: [[authHeader]]
Content-Type: application/json

Post Response 스크립트:

// 응답에서 access_token 추출 → 환경변수 + 다음 모듈로 전달
const out = getOutput();
setEnv('ACCESS_TOKEN', out.data.access_token);     // 영구 저장
setOutput('token', out.data.access_token);          // 다음 모듈 [[token]]

시나리오 B: 응답 데이터를 변형 후 일부만 다음 모듈로 보내기

// Post Response
const users = getOutput().items.filter(u => u.active);
setOutput('activeUsers', users);   // 객체 배열 저장
setOutput('count', users.length);

시나리오 C: 이전 모듈이 단일 객체를 보낸 경우 — getInput() / [[]] 동시 사용

이전 모듈의 OUTPUT이 다음과 같다고 가정:

{ "name": "lee dohan", "age": 21 }

Pre Request 스크립트에서 두 가지 방식 모두 가능:

// 방식 1 — getInput()으로 전체 JSON 받아 가공
const u = getInput();
console.log(u.name, u.age);                  // "lee dohan" 21
setInput('greeting', `Hello ${u.name}!`); // 가공된 값을 [[greeting]]으로

설정 (가운데 컬럼) — 스크립트 없이도 [[name]] 사용 가능:

POST  https://api/users
Content-Type: application/json
body:
{
  "userName": "[[name]]",       // → "lee dohan" 자동 해석
  "userAge": "[[age]]",         // → "21"
  "hello": "[[greeting]]"      // → "Hello lee dohan!" (Pre에서 setInput)
}
요점
getInput() 으로 전체 JSON을 JS 객체로 받아 자유롭게 탐색·가공할 수 있고, 동시에 [[키]] 표기로도 upstream 필드를 직접 참조할 수 있습니다. 스크립트를 쓰지 않는 단순 케이스는 [[키]] 만으로 충분.

8리스크 & 결정 보류 사항

스크립트 샌드박스 수준
AsyncFunction은 renderer 프로세스에서 실행되므로 window, document, fetch 등 전역에 접근 가능합니다. 사용자가 직접 작성하는 코드라 신뢰가 전제됩니다. iframe·Worker 격리는 이번 범위에서 제외. 필요 시 후속 작업으로 추가 가능.
번들 크기 영향
Monaco는 약 2–3MB 추가됩니다. Electron 앱이라 사용자 체감 부담은 적지만 빌드 시간이 다소 증가할 수 있습니다. 웹 워커 기반 lazy load로 영향 최소화 예정.
기존 [[]] 의미 변경
현재 [[key]]는 "현재 row 필드"만 의미했지만, 변경 후에는 "moduleVars(상위 setOutput) ∪ row 필드"로 확장됩니다. 키 충돌 시 script값이 우선 → 기존 워크플로우에서 충돌하는 키가 있다면 동작이 달라질 수 있음. 단, 현재 단일 사용자·작은 워크플로우 규모라 영향 미미할 것으로 판단.
setEnv 자동 저장
Post Response의 setEnv는 실행 즉시 환경변수에 영구 저장됩니다 (사용자 확인 절차 없음). 의도하지 않은 덮어쓰기가 발생하지 않도록 로그 패널에 변경 내역을 명시할 예정입니다.
