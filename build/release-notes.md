# a8a 업데이트 안내

이번 버전은 캔버스 실행 상태, 모듈 설정 창, SELECT/API 입력 선택, 실행 로그 가시성을 함께 개선한 Windows 릴리즈입니다.

- 캔버스 실행 버튼이 `실행`, `실행중`, `초기화` 상태를 명확하게 표시합니다.
- `Ctrl+Enter`로 현재 실행 상태에 맞춰 초기화 후 재실행할 수 있고, 실행 중 버튼을 누르면 중지 여부를 확인합니다.
- 실행 중에는 로딩 인디케이터를 표시하고, 초기화 버튼은 실행 가능한 액션임을 더 분명하게 보이도록 조정했습니다.
- PRE REQUEST와 POST RESPONSE 스크립트의 콘솔 로그가 모듈 콘솔 탭과 실행 로그에 유지됩니다.
- 실행 로그에서 API 모듈의 실제 요청을 cURL 명령어로 확인할 수 있습니다.
- DATA, SELECT, API, BRANCH, END 설정 창에서 전체화면 확대를 사용할 수 있습니다.
- 모듈 설정 창에서 `Ctrl+S` 또는 macOS `Cmd+S`로 저장할 수 있고, 단축키 저장 후 창 닫기 여부를 설정에서 관리할 수 있습니다.
- SELECT JSON 선택 창은 전체화면 확대, 이전 선택 값 기억, JSON 트리 펼침/접힘 상태 유지를 지원합니다.
- BRANCH 값이 고정 TRUE 또는 FALSE일 때 모듈의 TRUE/FALSE 포트가 더 분명하게 강조됩니다.
- API 모듈 생성 시 `Content-Type: application/json` 헤더가 기본으로 추가됩니다.
- API Header/Query 입력 행에서 Value 입력 영역을 더 넓게 조정했습니다.
- `{{ }}`, `[[ ]]`, `<< >>` 템플릿에서 간단한 숫자식과 문자열식을 사용할 수 있습니다.
- `[[amount * -1]]` 같은 INPUT 표현식의 미리보기 값이 실제 전송 값과 일치하도록 수정했습니다.
- SELECT 배열 출력이 API 모듈로 들어올 때 각 항목을 한 건씩 순차 요청하도록 복구했습니다.
- 템플릿 표현식은 허용된 산술, 경로 접근, 문자열 메서드, 변환 함수만 실행하는 제한 평가 방식으로 처리합니다.
- API INPUT 매핑은 조건에 따라 SELECT 입력 노드가 바뀌어도 노드 ID에 묶이지 않는 `output[0]` 경로를 저장합니다.
- JSON 트리에서 값을 선택한 뒤에도 API INPUT 매핑 경로를 직접 수정할 수 있습니다.
